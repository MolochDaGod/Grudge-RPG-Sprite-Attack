Deploy a New PvP Server

Option 1: Railway (Fastest)

Fork the server repo

Go to github.com/MolochDaGod/grudge-pvp-server and fork it.

Create Railway project

npm install -g @railway/cli
railway login
railway init --name my-grudge-pvp
railway link

Deploy

railway up --detach

Get your URL

railway domain

You'll get something like my-grudge-pvp-production.up.railway.app

Point the game client to your server

Set the Vercel environment variable:

VITE_PVP_SERVER_URL=https://my-grudge-pvp-production.up.railway.app

Option 2: Docker (Any VPS)

Clone and build

git clone https://github.com/MolochDaGod/grudge-pvp-server.git
cd grudge-pvp-server
docker build -t grudge-pvp .

Run the container

docker run -d \
  --name grudge-pvp \
  --restart unless-stopped \
  -p 5000:5000 \
  -e PORT=5000 \
  grudge-pvp

Verify it's running

curl http://localhost:5000/health
# {"status":"ok","rooms":0,"uptime":...}

Option 3: Cloudflare + VM (Production)

Deploy Docker on a VM (Hetzner, DigitalOcean, AWS, etc.)

Same Docker commands as Option 2.

Add Cloudflare DNS record

In your Cloudflare dashboard for grudge-studio.com:

Type: A
Name: fighter-api
Value: [YOUR_VM_IP]
Proxy: ON (orange cloud)

Update the game client

VITE_PVP_SERVER_URL=https://fighter-api.grudge-studio.com

Cloudflare handles SSL and proxies WebSocket natively — no extra config needed.

Option 4: Node.js Direct (No Docker)

git clone https://github.com/MolochDaGod/grudge-pvp-server.git
cd grudge-pvp-server
npm install
PORT=5000 node index.js

Scaling Beyond 1 Server

A single server handles ~500 concurrent rooms. For more:

Add Redis Adapter

npm install @socket.io/redis-adapter redis

Then in index.js:

import { createAdapter } from "@socket.io/redis-adapter";
import { createClient } from "redis";

const pub = createClient({ url: process.env.REDIS_URL });
const sub = pub.duplicate();
await Promise.all([pub.connect(), sub.connect()]);
io.adapter(createAdapter(pub, sub));

Then run multiple server instances behind a load balancer (nginx, Cloudflare, or Railway scaling).

Connecting the Game Client

The game client uses the usePvP hook (client/src/hooks/usePvP.ts) which connects via socket.io:

import { io } from "socket.io-client";

const socket = io("https://YOUR_SERVER", {
  path: "/pvp",
  transports: ["websocket", "polling"],
});

To point the deployed game at your server, set the Vercel env var:

# Vercel dashboard → Settings → Environment Variables
VITE_PVP_SERVER_URL = https://your-server.example.com

Then redeploy. The client reads this at build time.

Troubleshooting

Connection Failed

Check the server is running: curl /health. Verify CORS allows your domain. Check WebSocket support on your proxy.

Both Players Control P1

Fixed in latest build. Each client controls their assigned slot (p1/p2). Remote inputs use separate key refs.

Room Not Found

Rooms auto-delete after 10 minutes if not started. Codes are 4 uppercase characters.

Lag / Desync

Input relay architecture means both clients run the same simulation. High latency causes visible delay but not desync. Use a server geographically close to players.
